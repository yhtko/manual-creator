# 業務手順書自動作成ツール

Scribe のような共有機能やクラウド管理機能は持たず、日本企業向けの業務手順書を Word / PowerPoint に最短で出力するためのツールです。

## 構成

- `extension/`: Manifest V3 Chrome 拡張。サイドパネルから記録を開始し、タブ録画とクリックイベントを保存します。
- `web/`: React 製の編集・出力アプリ。記録データを読み込み、手順編集、画像マーキング、Word / PowerPoint 出力を行います。

## 設計メモ

- [録画ファースト方式の設計](docs/recording-first-design.md)

現行版は録画を一次ソースにし、クリック時刻からスクリーンショットを切り出す方式です。モーダル消失や画面遷移タイミングのズレを、クリックごとの静止画撮影ではなく録画フレーム選択で吸収します。

## 開発

```bash
npm install
npm run dev
```

## GitHub Pages

WebアプリはGitHub Pagesで公開します。ローカルホストは開発時だけ必要です。

```text
https://yhtko.github.io/manual-creator/
```

GitHubリポジトリのPages設定は `GitHub Actions` を選択してください。`main` ブランチへpushすると、`.github/workflows/pages.yml` が自動でビルドと公開を行います。

## Chrome 拡張の読み込み

1. Chrome で `chrome://extensions` を開きます。
2. デベロッパーモードを有効にします。
3. 「パッケージ化されていない拡張機能を読み込む」から `extension/` を選択します。
4. 拡張アイコンをクリックすると、右側にサイドパネルが開きます。

## 記録データの使い方

1. サイドパネルで「記録開始」を押します。
2. 対象ページでクリック操作を行います。サイドパネルは開いたまま記録を継続できます。
3. サイドパネルで「記録停止して保存」を押します。
4. `manual-project/` に `events-YYYYMMDD-HHMMSS.json` と `recording-YYYYMMDD-HHMMSS.webm` がダウンロードされます。
5. GitHub PagesのWebアプリを開き、`events-YYYYMMDD-HHMMSS.json` を読み込みます。
6. 続けて `recording-YYYYMMDD-HHMMSS.webm` を読み込むと、各クリック時刻のスクリーンショットが自動生成されます。
7. 作成日、作成者、目的、説明文を編集します。
8. Word / PowerPoint を出力します。
9. 次回編集用に「編集JSON保存」から `作業手順書-project-YYYYMMDD-HHMMSS.json` を保存できます。

## Word出力

WebM録画はクリック時点のスクリーンショット抽出素材として使用し、最終成果物はWordを主軸にします。

- 表紙
- 作成日、作成者、目的、対象者、前提条件、完了条件の概要表
- 手順一覧
- 各ステップの画像と説明文を横並びにしたカード形式
- クリック位置が分かる赤丸・ステップ番号付き画像
- 日本語 / 英語の項目名切替

動画はWord出力には含めません。

## PowerPoint出力

PowerPointは説明会や画面共有向けの補助出力です。

- 表紙スライド
- 概要スライド
- 1ステップ/スライド、2ステップ/スライド、4ステップ/スライドを選択可能
- 1ステップ表示では画像を大きく配置し、右側に説明文と画面情報を表示
- 複数ステップ表示ではカード形式で画像と説明文を整理
- 日本語 / 英語の項目名切替

GIF変換、MP4変換、HTML出力、ZIP出力は初期版の対象外です。録画済みのWebMはクリック時点のスクリーンショット抽出素材として使用します。

## 編集済みJSON

Webアプリで文言修正、ステップ削除、並び替え、作成日、作成者、目的、対象者、前提条件、完了条件を編集した後、「編集JSON保存」を押すと編集済みJSONを保存できます。

- 編集済みJSONは次回編集時の正本として使います。
- 画像データと動画データはJSONに埋め込みません。
- 次回は編集済みJSONを「JSON読込」で読み込み、対応する `recording-*.webm` を「動画読込」で読み込むと画像を再生成できます。
- 複数記録を結合している場合も、編集済みJSON内に録画ファイル名の情報を保持します。

## AI説明文の土台

各ステップを単独でAIに渡すのではなく、全工程の流れ、前後ステップ、DOM情報、画面タイトル、クリック時刻、クリック前後の動画区間をまとめた `AI用JSON` を出力できます。

- `enhanced_description`: 新人向けに肉付けした説明
- `purpose`: その操作を行う目的
- `check_point`: 操作後に確認すべきこと

Webアプリ上では各ステップに「肉付け説明」「操作目的」「確認ポイント」を入力できます。これらの項目は編集済みJSONに保存され、Word出力にも反映されます。

AIが返したJSONは「AI結果JSON読込」から取り込めます。`steps` 配列、配列そのもの、または単一ステップJSONを読み込み、`step_no` をキーにして各ステップへ反映します。

APIキーはChrome拡張機能へ埋め込みません。将来的にAI連携する場合は、ユーザーがWebアプリの設定画面で入力する方式、または社内プロキシ経由の方式を検討します。

WebアプリにはAI設定欄があります。

- Provider
- Model
- API Key
- この端末に保存

APIキーは標準ではブラウザのメモリ上だけに保持し、ページを閉じると消えます。「この端末に保存」を選んだ場合のみ、利用中のブラウザのlocalStorageへ保存します。

## 説明文生成

DOM情報から初期説明文を生成します。

- 録画内容から、目的、対象者、前提条件、完了条件の初期値を推定
- 保存、検索、追加、編集、削除などのボタン文言を判定
- kintoneの「レコードの一覧」「レコードの詳細」画面を考慮
- 数字ID、メールアドレス、日付などの具体値は汎用表現へ寄せる

## 互換入力

以前の方式で保存した `manual-project.json` と `step_XXX.png` も読み込めます。その場合は、WebアプリでJSONとPNGを選択してください。

## 複数記録の結合

出力済みファイルを手作業で結合するのではなく、Webアプリ上で記録データを追加してから1つのWordとして出力します。

1. 1つ目の `events-*.json` を「JSON読込」で読み込みます。
2. 1つ目の `recording-*.webm` を「動画読込」で読み込みます。
3. 2つ目以降の `events-*.json` を「JSON追記」で追加します。
4. 追加した記録の `recording-*.webm` を「動画読込」で読み込みます。
5. 最後に「Word出力」を押します。
